    安装&配置

  • step1：将连线连接好，接入电源。但是确保不要接入USB，确保不要接入USB，确保不要接入USB。重要的事情说3遍

  • step2：双击打开KinectSDK-v2.0_1409-Setup.exe

  • step3：SDK程序安装完成之后，这个时候插入USB3.0，然后就是等待它自动加载驱动直到完成

##4.动作确认
主要有几种方法：

  • kinect的说明书上面写的一样，正常的有白色的灯和3个LED指示灯会亮，kinect的连接转换器的指示灯正常的时候是亮白色，接电源但是没有连接USB3.0的情况下是黄色。
  • 查看设备管理器是否有Kinectsensor这一项
  • Kinect的麦克风，查看设备管理器是否有Kinect for Windows USB Audio这一项

##5.问题对策（安装没有问题的请略过此章节）
大家最不想看到的就是，按照上面写的安装好了，却运行不了或者出错这是怎么回事（此处应有捉急的图片）
木办法，人生不如意之事十有八九，俺刚开始安装的时候也是遇到遇到了问题这里跟大家哭诉一下（血的教训），刚开始安装的时候没有好好的进行动作确认（让我先哭一会er）以致于学习了可视画像，深度图像之后，却怎么也显示不了红外线画像，进展一下子就停住了，后来有一位前辈（很感谢他）告诉我用Vistual Studio 进行debug的方法（谁让我一直用eclipse呢，因为学校一直就是eclipse，估计是因为 1.开源不用花钱买license吧，貌似 Vistual Studio很贵哒，2.一般都是用linux上课，二者更合适），发现红外摄像头获取的图像的矩阵返回的都是0，后来在Troubleshooting里找到了解决的办法，当时想了千万种可能（此处运用夸张的修辞手法）怎么也没想到是因为用的英伟达的显卡用的驱动程序旧了，我当时特想对这台电脑以前的使用者说：你都什么不用更新吗？，话说回来，大家的力量是无穷的，要不是有这些资料的提醒还不知道要搞到什么时候呢！

我们开看看这里是MSDN的Troubleshooting
我就遇到过上面的英伟达显卡的驱动问题，其他的虽然没什么发言权，暂且在这里搬运并罗列一下（基本来自于官网），具体的请去上面的链接去淘

  • 如果不显示任何输出

    • 请确认sensor， 电源，USB 3.0连接是否正常
    • 通过设备管理器确认是否只运行了一个kinect的服务
    • 可能需要重新插拔一下设备
    • 如果有其他的USB设备连接在电脑上，请移除它
    • 确定你的电脑是否符合kinect关于硬件以及软件的要求
    • 再启动吧

  • 红外线数据流返回的都是0（这时候红外图像的例子会显示黑屏，但是可视图像和深度图像正常）

    • 这时候肯能就是英伟达GPU的显卡的驱动旧了，毕竟是2014年9月出的SDK，这之前的显卡驱动可能不支持 ，所以少年哟，去下载最新的WHQL驱动吧

  • 声音的程序例崩溃了？

    • 确定你是否安装了x86的运行环境

  • USB 3.0

    • 如果添加通过安装PCLe来使用USB 3.0功能，确认支持Gen-2.你可能需要尝试不同的PCL-e 插槽来找到能达到kinect 带宽要求的那个。将适配器插入到图形用的插槽最合适。

    • 如果你收到系统资源不足的警告信息，说明你其他的设备占用了控制器的带宽，从控制器移除其他设备，比如webcam。如果不能解决，请重启（重启真是万能的！）。

    这是MSDN社区的Troubleshooting 比较详细，需要的同学请戳链接。
    原文给了两个链接有兴趣的同学自己去看一下喽： Release notes和Known issues
